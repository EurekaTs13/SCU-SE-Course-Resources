## 🎯 贡献途径 (How to Contribute)

### 1️⃣ 贡献资料 ⭐

这是最常见也最有价值的贡献方式！你可以提供以下类型的资料：

#### 📝 复习资料
- 期末期中复习重点、复习心得、知识点总结、思维导图、公式速查表等
- **小贴士**：如果能用 Markdown 格式就更好了！方便后续编辑和维护 (๑•̀ㅂ•́)و✧

#### 💯 历年试题
- 期末考试、期中考试、小测等试卷回忆版
- 标准答案或解题思路（不仅有题，最好还能有解析）
- **注意**：请确保你上传的试题**不属于机密内容**，不是老师明确禁止外传的题库 ⚠️

#### 💻 课程设计 & 实验报告
- 课程大作业源码、实验报告、需求分析文档
- 个人优化版本、踩坑经验总结
- **小提示**：请确保代码可以运行，最好附上运行环境说明和简单的 README 📄

#### 📚 其他资源
- 学习路线图、推荐书籍、在线资源链接
- 工具使用教程（如 IDEA 配置、Git 命令等）
- 客观的课程评价和选课建议

**需要注意** 📢：
- **视频类资源**：请只提供链接，不要直接上传大文件
- **单个文件超过 100 MB**：请以网盘链接的方式给出
- **版权资源**：严禁上传教师课件！详见[负面清单](./NEGATIVE_LIST.md)

---

## 📋 贡献规范 (Contribution Standards)

由于目前仓库内容较多，为了保持整洁和方便检索，请严格按照规范进行贡献资料 (｡•̀ᴗ-)✧

### 🚫 需要避免的问题

**请务必不要上传以下内容** ❌：

1. **涉及隐私的内容**：电话号码、家庭住址、身份证号等个人信息 🔒
2. **主观攻击性内容**：对老师或课程的纯粹情绪化吐槽（客观评价是可以的）😤
3. **版权侵犯内容**：**严格禁止上传任何老师的课件**（PPT/讲义/电子书等）⚠️
4. **违规违法内容**：详见[《负面清单》](./NEGATIVE_LIST.md) 📜

### 📁 文件命名规范

我们实行 **"文件夹一级命名负责制"**。简单来说：

> **我们只强制要求你上传的根文件夹（第一层）遵循规范** ✅  
> 文件夹内部的结构，你可以根据项目实际情况自由安排，但请做到 **"见文知意"**！

**命名格式：**

```text
你的网名_上课年份-学期_分数
```

**各部分说明：**
- **你的网名**：你的昵称或代号（请脱敏，不要使用真实姓名），如 `Karry`、`AliceWang`、`DevMaster` 等
- **上课年份-学期**：你上这门课的年份和学期，格式为 `YYYY-Fall` 或 `YYYY-Spring`
  - `Fall` 表示秋季学期（大一上、大二上、大三上、大四上）
  - `Spring` 表示春季学期（大一下、大二下、大三下、大四下）
  - 例如：`2023-Fall`（2023年秋季）、`2024-Spring`（2024年春季）
- **分数**：你这门课的最终成绩，如 `95`、`A`、`优秀` 等
  - 如果不想公开分数，可以用 `XX` 代替

**✅ 好的示例：**
```
Karry_2023-Fall_95
AliceInSCU_2024-Spring_A
DevMaster_2022-Fall_XX
TechGeek_2023-Spring_90
```

**❌ 不好的示例：**
```
张三_2023_95             # 使用了真实姓名
我的资料                 # 没有按规范命名
2022Java作业            # 格式混乱
NewFolder(1)            # 无意义命名
```

### 📂 文件夹内部结构建议

虽然我们不强制要求内部结构，但还是给出一些建议 (•ω•)

**对于代码项目：**
```
Karry_2023-Fall_95/
├── src/                    # 源代码
├── doc/                    # 文档
├── README.md               # 项目说明（强烈推荐！）
└── screenshots/            # 运行截图
```

**对于笔记/试题类：**
```
AliceInSCU_2024-Spring_A/
├── Chapter1_Matrix.md
├── Chapter2_VectorSpace.md
└── FormulasCheatSheet.pdf
```

**核心原则：**
1. **层次清晰**：不要把所有文件堆在一个文件夹里 🗂️
2. **命名规范**：文件名要有意义，避免 `新建文本文档.txt`、`未命名1.docx` 这样的名字 📝
3. **附带说明**：如果项目稍复杂，请写一个 `README.md`，说明如何使用 📖

---

### ✅ 贡献前的检查清单

在提交资料前，请花 1 分钟检查以下几点：

- [ ] **合规合法**：资料是否合规合法，没有违反[负面清单](./NEGATIVE_LIST.md) ✔️
- [ ] **隐私脱敏**：是否已去除敏感个人信息（姓名、学号、电话等）🔐
- [ ] **避免重复**：检查课程文件夹下是否已有类似资料，避免重复上传 🔍
- [ ] **文件完整**：代码能运行？文档能打开？链接有效？✨

**全部打钩？那就放心提交吧！** (ง •̀_•́)ง

## 🚀 贡献方法和详细步骤

我们提供了多种贡献方式，你可以根据自己的熟悉程度选择 (•̀ω•́)✧

### 方式 A：Email / Issues 📧

GitHub 原为代码托管网站，对于不熟悉 GitHub 工作流程的同学来说，学习成本可能略高 (´･ω･`)

如果你不太熟悉 Git，或者只是想快速贡献资料，可以使用 **Email** 或 **Issue** 的方式 ✉️

**贡献步骤：**

1. **检查资源**：确认你要贡献的资源是否已经存在，是否符合仓库收录范围 🔍
2. **资料自查**：按照上述规定检查资源有无隐私内容、违规内容、版权问题等 ✔️
3. **发送资料**：使用 Email 或者 Issue 把资源发送给我们（建议压缩为 ZIP 文件）📦

**具体方法：**

#### 📮 通过 Email
- **邮箱地址**：`KarryRenKai@outlook.com`
- **邮件标题**：`[资源贡献] 课程名称 - 资料类型`
- **邮件正文**：请简单说明资料的内容、年份、注意事项等

#### 💬 通过 Issues（推荐）
1. 进入 [仓库首页](https://github.com/YourRepo/SCU-SE-Course-Resources)
2. 点击 **Issues** 选项卡
3. 点击 **New Issue** 按钮
4. 选择"贡献资源"Issue 模板，点击 **Get started**
5. 根据提示上传资源文件（或提供网盘链接），点击 **Submit new issue**

仓库维护者会将你发送的资源按照规范添加到仓库中，并在 Contributors 中感谢你 (ﾉ◕ヮ◕)ﾉ*:･ﾟ✧

---

### 方式 B：Pull Request（推荐⭐⭐⭐）

如果你熟悉 GitHub 的工作流程，**强烈推荐**使用 Pull Request 的方式直接贡献 (๑•̀ㅂ•́)و✧

**为什么推荐 PR？**
- 🚀 **效率更高**：你可以直接把资料放到合适的位置，维护者只需审核即可
- 📚 **学习技能**：掌握 GitHub 工作流是软件工程师的必备技能
- 🌟 **成就感满满**：亲手提交 PR 并被 Merge，会让你有满满的成就感！

---

#### � PR 提交完整教程

**整体流程：**
```
Fork → Clone → Add Files → Commit → Push → Pull Request → Review → Merge
```

**详细步骤：**

##### 1️⃣ Fork 本仓库 🍴

1. 登录你的 GitHub 账号（如果还没有，请先去 [GitHub](https://github.com) 注册）
2. 进入 [本仓库首页](https://github.com/YourRepo/SCU-SE-Course-Resources)
3. 点击右上角的 **Fork** 按钮
4. 确认 Fork，稍等片刻，你的账号下就有了一个同名的仓库副本 ✨

之后你会看到仓库名下方显示：`forked from YourRepo/SCU-SE-Course-Resources`

---

##### 2️⃣ Clone 到本地 💻

打开你的终端（Windows 用 Git Bash，Mac/Linux 用 Terminal），执行：

```bash
git clone https://github.com/你的用户名/SCU-SE-Course-Resources.git
cd SCU-SE-Course-Resources
```

这样，仓库就被下载到你的本地电脑了 📥

---

##### 3️⃣ 添加你的资源 📂

1. **找到对应的课程目录**，比如 `必修/02_大一下/Java程序设计/`
2. **创建你的文件夹**，记得遵守命名规范！比如：
   ```
   2022_Java_Proj_SpaceInvaders_YourName/
   ```
3. **把你的文件放进去**：代码、文档、README……
4. **最后检查一遍**：
   - 有没有个人隐私信息？🔒
   - 代码能正常运行吗？▶️
   - 有没有违规内容？⚠️

---

##### 4️⃣ Commit 提交 💾

回到终端，执行以下命令：

```bash
# 查看修改了哪些文件
git status

# 添加所有修改到暂存区
git add .

# 提交修改，并写上清晰的提交信息
git commit -m "feat: add 2022 Java SpaceInvaders project"
```

**关于 Commit Message：**
- 建议使用格式：`类型: 简短描述`
- 常用类型：`feat`（新增）、`fix`（修复）、`docs`（文档）、`refactor`（重构）

---

##### 5️⃣ Push 到远程仓库 🚀

```bash
git push origin main
```

（如果你创建了新分支，把 `main` 替换成你的分支名）

现在，你的修改已经上传到**你自己 Fork 的仓库**了 ☁️

---

##### 6️⃣ 发起 Pull Request 🔀

1. 回到 GitHub 页面，进入你 Fork 的仓库
2. 你会看到一个黄色提示条，上面写着 **"Compare & pull request"**，点击它
3. **填写 PR 信息：**
   - **标题**：简洁明了，如 `Add 2022 Java SpaceInvaders Project`
   - **描述**：简单介绍你上传了什么，有什么特点
4. 点击 **"Create pull request"**

恭喜！🎉 你的 PR 已经提交，接下来就等待仓库维护者审核了 (ﾉ◕ヮ◕)ﾉ*:･ﾟ✧

---

##### 7️⃣ 等待 Review 👀

维护者会检查你的 PR：
- 是否符合命名规范？✅
- 有没有违规内容？⚠️
- 文件是否完整？📦

如果有问题，维护者会留下评论，你只需要按照建议修改后再次 Push，PR 会自动更新 🔄

如果一切顺利，你的 PR 会被 **Merge（合并）** 到主仓库，你就正式成为了 **Contributor** 之一！🌟🌟🌟

#### � 常见问题 FAQ

**Q: 我不会用 Git Bash，可以用网页版操作吗？**  
A: 可以的！你可以直接在 GitHub 网页上进行操作：
1. Fork 仓库后，进入对应目录
2. 点击右上角 **Add file** → **Upload files**
3. 拖拽文件上传，填写 Commit 信息
4. 回到仓库首页，点击 **Contribute** → **Open pull request**

**Q: 遇到分支冲突怎么办？**  
A: 尝试搜索 "Git 解决冲突"，或者在 Issue 中求助，我们会帮你 (•̀ω•́)✧

**Q: 提交 PR 后多久会被审核？**  
A: 通常在 3-7 天内，请耐心等待。如果超过 7 天没有回复，可以在 PR 下留言提醒 �
